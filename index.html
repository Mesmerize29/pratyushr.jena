Signature projects

LLM security framework

Interaction centric LLM security framework

This work started as a simple concern. Most LLM security talk felt like a list of party tricks. Prompt this, block that. Nothing about how the whole system behaves over time.

I reframed it as an interaction chain. Inputs, internal processing, outputs. At each stage I mapped failure patterns, attacks, and what a realistic defense would look like if it had to live inside a product.

The result is an interaction centric taxonomy and a layered defense pipeline, structured as an IEEE style review and a blueprint for future tools.

Autonomous platform

Hexacopter and cyber physical behavior

On paper, the hexacopter is a hardware project. In practice it is a computer science problem. Modes, states, transitions, and what happens when the world acts like a bug report.

Using Navio2, LiDAR, TF Mini, and Mission Planner I worked on how the system should react when ranges drop, wind changes, or telemetry lies. The goal was not just to fly but to stay explainable.

It forced me to express behavior as clear states and transitions and to use telemetry and logs as first class tools for debugging reality.

RealSense vial classification

Vision guided vial classification and mapping

In labs, nothing is perfectly aligned. Lighting shifts, holders move, caps are missing. I built a RealSense based system that accepts this instead of fighting it.

The system detects cap colors, segments vials into four holders, and creates a clean map for a future robot. Black caps, missing caps, and empty spots are handled in one consistent way so the controller does not have to guess.

The real win is the interface. A bridge between noisy pixels and simple data structures that robotic logic can trust.

How I think about systems

No one builds good systems by stacking features. I think in interactions, states, and how things behave when you let them run for a long time.

Interaction chains

I look at how information enters, moves, and leaves. Every stage can amplify or dampen risk, so every stage needs its own checks and structure.

Systems over components

A perfect module in a broken system is still a failure. I care more about the full loop than any single elegant function.

Nature inspired AI

Ecosystems solve stability and adaptation with simple local rules. I borrow those ideas when I design control, policies, and safety logic.

Safety before scale

I would rather have a smaller system I understand than a huge system that surprises me. Scale comes after I trust how it behaves.

Long term architecture

I try to design with future failure in mind. How will this system behave after months of weird data and human shortcuts.

Where this is heading

Short version: I want intelligent systems that actively help preserve and regenerate the world instead of quietly extracting from it.

My long term work aims to build intelligent systems capable of preserving, understanding, and regenerating ecosystems. That means AI models that are aware of their impact, robotic platforms that behave like responsible visitors, and architectures that keep humans accountable and in the loop.

The goal is not just smarter systems. It is systems that make it easier for people to do the right thing for the places we live in.
